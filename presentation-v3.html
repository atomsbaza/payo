🛡️ Security Architecture

Defense-in-depth for a fintech application

🔐 Link Integrity (HMAC-SHA256)

Every payment link is signed server-side. Tampered links are fully blocked — no soft warnings, no way to pay.

✅ Input Validation (Zod)

All payment data validated: Ethereum address format, supported tokens, amount range (0–1M), chainId = Base Sepolia.

🚦 Rate Limiting

API routes protected: 10 req/min for TX queries, 20 req/min for link creation. In-memory store, 429 + Retry-After.

🔒 HTTP Security Headers

CSP, X-Frame-Options: DENY, HSTS, nosniff, Referrer-Policy, Permissions-Policy — all via Next.js middleware.

🚫 Self-Payment Guard

Detects when payer === payee (case-insensitive). Pay button disabled with warning message.

💾 LocalStorage Validation

Schema validation on read. Invalid items filtered out. Corrupt JSON returns empty array

🚫 Tampered Link Blocking

Zero tolerance for modified payment links
